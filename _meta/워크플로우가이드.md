# 스터디 워크플로우 가이드

## 멤버가 하는 일 (간단!)

### 1. 미션 작성
- `00_주차별미션/Week_0N/` 에서 본인 파일 열기
- 내용 작성 후 저장

### 2. 개념 정리 (선택)
- `01_개념/Week_0N/` 에 새 파일 만들기
- 파일명: `Week_0N_개념주제.md` (예: `Week_01_AgentLoop개념.md`)
- `_templates/개념_템플릿.md` 참고

### 3. 인사이트 작성 (선택)
- `02_인사이트/Week_0N/` 에 새 파일 만들기
- 파일명: `Week_0N_인사이트주제.md` (예: `Week_01_멀티에이전트_활용법.md`)
- `_templates/인사이트_템플릿.md` 참고

### 4. Push
```bash
git pull origin main
git add .
git commit -m "Week_01 미션 제출 - 닉네임"
git push origin main
```

## 폴더 안내

| 폴더 | 누가 쓰나 | 설명 |
| --- | --- | --- |
| 00_주차별미션/ | 멤버 | 주차별 과제 |
| 01_개념/ | 멤버 | 배운 개념 정리 |
| 02_인사이트/ | 멤버 | 생각/의견/아이디어 |
| _분석/ | AI 자동 | 통합 분석 리포트 |
| _프로젝트제안/ | AI 자동 | 프로젝트 제안 |
| _아카이브북/ | AI 자동 | 학습 아카이브 |
| _현황/ | AI 자동 | 진행 현황 |

> `_` 로 시작하는 폴더는 AI가 자동 생성합니다. 직접 수정하지 않아도 됩니다.

## 운영자(다다)가 하는 일

멤버들이 push한 후 Claude Code에서 순서대로 실행:

```
/analyze 1        ← 1주차 통합 분석 + 현황 업데이트
/propose          ← 프로젝트 제안 (필요할 때만)
/archive          ← 아카이브 북 갱신
```

그 후 결과를 커밋 & push:
```bash
git add .
git commit -m "Week_01 분석 완료"
git push origin main
```
