# 스터디 워크플로우 가이드

## 최초 설정 (한 번만!)

### 1. Git 설치 확인
```bash
git --version
```
없으면 https://git-scm.com 에서 설치

### 2. 레포 클론
```bash
git clone https://github.com/selfishclub-all/aaa.git
cd aaa
```

### 3. Git 사용자 설정
```bash
git config user.name "본인닉네임"
git config user.email "본인이메일"
```

### 4. GitHub 인증 (둘 중 하나 선택)

**방법 A - Personal Access Token (추천)**
1. GitHub 로그인
2. Settings > Developer settings > Personal access tokens > Generate new token
3. repo 권한 체크 후 생성
4. push 할 때 비밀번호 대신 이 토큰 입력

**방법 B - SSH 키**
```bash
ssh-keygen -t ed25519
cat ~/.ssh/id_ed25519.pub
```
출력된 키를 GitHub > Settings > SSH and GPG keys > New SSH key에 추가

### 5. Obsidian에서 vault 열기
Obsidian > Open folder as vault > 클론한 aaa 폴더 선택

## 멤버가 하는 일 (간단!)

### 1. 미션 작성
- `00_주차별미션/Week_0N/` 에서 본인 파일 열기
- 내용 작성 후 저장

### 2. 개념 정리 (선택)
- `01_개념/` 에 새 파일 만들기
- 파일명: `주제.md` (예: `AgentLoop개념.md`)
- `_templates/개념_템플릿.md` 참고

### 3. 인사이트 작성 (선택)
- `02_인사이트/` 에 새 파일 만들기
- 파일명: `주제.md` (예: `멀티에이전트_활용법.md`)
- `_templates/인사이트_템플릿.md` 참고

### 4. Push
```bash
git pull origin main
git add .
git commit -m "Week_01 미션 제출 - 닉네임"
git push origin main
```

> 충돌이 나면 `git pull origin main` 먼저 하고 다시 push

## 운영자(다다)가 하는 일

멤버들이 push한 후 Claude Code에서 순서대로 실행:

```
/analyze 1        ← 1주차 통합 분석 + 현황 업데이트
/propose          ← 프로젝트 제안 (필요할 때만)
/archive          ← 아카이브 북 갱신
```

그 후 결과를 커밋 & push:
```bash
git add .
git commit -m "Week_01 분석 완료"
git push origin main
```
