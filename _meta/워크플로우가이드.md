# 스터디 워크플로우 가이드

## 최초 설정 (한 번만!)

### 1. Git 설치 확인
```bash
git --version
```
없으면 https://git-scm.com 에서 설치

### 2. 레포 클론
```bash
git clone https://github.com/selfishclub-all/aaa.git
cd aaa
```

### 3. Git 사용자 설정
```bash
git config user.name "본인닉네임"
git config user.email "본인이메일"
```

### 4. GitHub 인증 (둘 중 하나 선택)

**방법 A - Personal Access Token (추천)**
1. GitHub 로그인
2. Settings > Developer settings > Personal access tokens > Generate new token
3. repo 권한 체크 후 생성
4. push 할 때 비밀번호 대신 이 토큰 입력

**방법 B - SSH 키**
```bash
ssh-keygen -t ed25519
cat ~/.ssh/id_ed25519.pub
```
출력된 키를 GitHub > Settings > SSH and GPG keys > New SSH key에 추가

### 5. Obsidian에서 vault 열기
Obsidian > Open folder as vault > 클론한 aaa 폴더 선택

## 멤버가 하는 일

### 1. 미션 작성
- `00_주차별미션/Week_0N/` 에서 본인 파일 열기
- 템플릿 삽입: `Ctrl+P` → "템플릿 삽입" → `미션_Week_0N_닉네임` 선택
- 내용 작성 후 저장

### 2. 개념 정리 (선택)
- `01_개념/` 에 새 파일 만들기
- 파일명: `주제.md` (예: `AgentLoop개념.md`)
- 템플릿 삽입: `Ctrl+P` → "템플릿 삽입" → `개념_주제` 선택

### 3. 인사이트 작성 (선택)
- `02_인사이트/` 에 새 파일 만들기
- 파일명: `주제.md` (예: `멀티에이전트_활용법.md`)
- 템플릿 삽입: `Ctrl+P` → "템플릿 삽입" → `인사이트_주제` 선택

### 4. 브랜치 만들고 Push (터미널에서)
```bash
# 최신 내용 받기
git pull origin main

# 내 브랜치 만들기 (이름은 자유!)
git checkout -b 비비안/week02-미션

# 작업 후 push
git add .
git commit -m "Week_02 미션 제출 - 비비안"
git push origin 비비안/week02-미션
```

### 5. PR(Pull Request) 만들기
1. push하면 터미널에 뜨는 링크 클릭, 또는
2. GitHub 레포 페이지에서 "Compare & pull request" 버튼 클릭
3. 제목 적고 "Create pull request"
4. 다다가 확인 후 머지!

### 6. 다음 작업 시작할 때
```bash
git checkout main
git pull origin main
git checkout -b 새브랜치이름
```

> 브랜치 이름 예시: `비비안/week02-미션`, `띵끄/개념정리`, `잽/갤러리추가`

## Push 방법 (브랜치 워크플로우)

### 전체 흐름
```
main에서 최신 받기 → 내 브랜치 만들기 → 작업 → push → PR 만들기 → 다다가 머지
```

### 1. 작업 시작
```bash
cd aaa
git checkout main
git pull origin main
git checkout -b 닉네임/작업내용
```

### 2. 작업 후 push
```bash
git add .
git commit -m "Week_02 미션 제출 - 비비안"
git push origin 닉네임/작업내용
```

### 3. PR 만들기
- push 후 터미널에 나오는 링크 클릭
- 또는 GitHub 레포 페이지에서 "Compare & pull request" 클릭
- 제목 적고 "Create pull request"

### 예시
```bash
# 비비안이 2주차 미션 제출
git checkout main
git pull origin main
git checkout -b 비비안/week02-미션
# Obsidian에서 작업...
git add .
git commit -m "Week_02 미션 제출 - 비비안"
git push origin 비비안/week02-미션
# → GitHub에서 PR 생성

# 띵끄가 개념 정리 추가
git checkout main
git pull origin main
git checkout -b 띵끄/agentloop-개념
# Obsidian에서 작업...
git add .
git commit -m "AgentLoop 개념 정리 - 띵끄"
git push origin 띵끄/agentloop-개념
# → GitHub에서 PR 생성
```

### 에러가 나면?
```bash
# "rejected" 에러 → main 최신화 후 다시
git checkout main
git pull origin main
git checkout 내브랜치
git merge main
git push origin 내브랜치

# 브랜치를 까먹고 main에서 작업했을 때
git stash
git checkout -b 닉네임/작업내용
git stash pop
git add .
git commit -m "메시지"
git push origin 닉네임/작업내용
```

## 운영자(다다)가 하는 일

멤버들이 push한 후 Claude Code에서 순서대로 실행:

```
/analyze 1        ← 1주차 통합 분석 + 벌금 계산 + 현황 업데이트
/propose          ← 프로젝트 제안 (필요할 때만)
/archive          ← 아카이브 북 갱신
```

그 후 결과를 커밋 & push:
```bash
git add .
git commit -m "Week_01 분석 완료"
git push origin main
```
