# 프로젝트 제안 스킬

전체 분석 데이터를 바탕으로 스터디 그룹에 맞는 프로젝트를 제안합니다.

## 실행 절차

### 1. 데이터 수집
- `_분석/주차별분석/` 의 모든 분석 리포트 읽기
- `_분석/멤버별분석/` 의 모든 멤버 프로필 읽기
- `_프로젝트제안/` 의 기존 제안 확인 (중복 방지)

### 2. 프로젝트 아이디어 도출
분석 데이터에서 다음을 찾아 프로젝트로 발전:
- **공통 관심사 클러스터** - 3명 이상이 관심을 보인 주제
- **시너지 조합** - 서로 다른 멤버의 개념+인사이트가 합쳐지면 강력한 프로젝트가 되는 경우
- **"해보고 싶은 것" 패턴** - 멤버들이 반복적으로 언급한 실험/시도
- **트렌드 연결** - 학습 내용과 현재 AI/ML 트렌드의 접점

### 3. 제안서 작성
각 프로젝트 제안마다 `_프로젝트제안/Proposal_NNN_제목.md` 생성:
- 제안 배경 (어떤 인사이트에서 출발했는지)
- 프로젝트 개요
- 기대 효과
- 필요 역량/도구
- 참여 추천 멤버 (프로필 기반)
- 실행 단계 (구체적인 스텝)
- 난이도, 예상 기간
- 관련 인사이트 wikilink 연결

### 4. 제안 목록 업데이트
- `_프로젝트제안/_제안목록.md` 에 새 제안 추가
- 기존 제안 목록은 유지하고 아래에 추가

## 제안 원칙
- 실현 가능한 프로젝트 위주 (스터디 그룹 규모에 맞게)
- 멤버 역량과 관심사를 반영
- 1~2주 안에 결과를 볼 수 있는 단위로 분할
- 한국어로 작성, Obsidian wikilink 활용
