# 제안 스킬

전체 분석 데이터를 바탕으로 스킬/플러그인 제안과 프로젝트 인사이트를 생성합니다.

## 실행 절차

### 1. 데이터 수집
- `_분석/주차별분석/` 의 모든 분석 리포트 읽기
- `_분석/멤버별분석/` 의 모든 멤버 프로필 읽기
- `_제안/스킬_제안.md` 기존 제안 확인 (중복 방지)
- `_제안/AAA_봇_인사이트.md` 기존 내용 확인

### 2. 스킬 제안 도출
분석 데이터에서 다음을 찾아 스킬/플러그인으로 발전:
- **반복 작업 패턴** - 자동화할 수 있는 작업 발견
- **공통 관심사 클러스터** - 3명 이상이 관심을 보인 주제
- **"해보고 싶은 것" 패턴** - 멤버들이 반복적으로 언급한 실험/시도
- **트렌드 연결** - 학습 내용과 현재 AI/ML 트렌드의 접점

제안 유형:
- **Claude Code 스킬** - 학습한 개념을 자동화하는 커스텀 스킬
- **Obsidian 플러그인** - vault 관리/시각화/자동화 플러그인
- **CLI 도구** - 반복 작업을 해결하는 터미널 도구

### 3. 결과 업데이트

**`_제안/스킬_제안.md`** 업데이트:
- 제안 목록 테이블에 새 항목 추가
- 상세 제안 섹션에 설명 추가
- 난이도, 관련 개념, 추천 멤버 포함

**`_제안/AAA_봇_인사이트.md`** 업데이트:
- 프로젝트 아이디어: 멤버 관심사 기반 새 프로젝트 제안
- 이렇게 하면 더 좋겠다: 현재 스터디/시스템 개선점
- 자유 생각: 분석에서 발견한 흥미로운 패턴이나 연결고리

## 제안 원칙
- 실현 가능한 것 위주 (스터디 그룹 규모에 맞게)
- 멤버 역량과 관심사를 반영
- 1~2주 안에 결과를 볼 수 있는 단위로 분할
- 한국어로 작성, Obsidian wikilink 활용
