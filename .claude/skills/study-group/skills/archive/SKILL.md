# 아카이브 북 스킬

인사이트와 분석 결과를 모아 Obsidian 마크다운 아카이브 북을 갱신합니다.

## 실행 절차

### 1. 데이터 수집
- `01_개념/` 의 모든 파일 읽기 (주차별 하위폴더 없음, 플랫 구조)
- `02_인사이트/` 의 모든 파일 읽기 (주차별 하위폴더 없음, 플랫 구조)
- `_분석/주차별분석/` 의 모든 분석 리포트 읽기
- `_프로젝트제안/` 의 모든 제안서 읽기

### 2. 목차 생성/업데이트
`_아카이브북/_목차.md` 생성 또는 업데이트:
```markdown
# 스터디 아카이브 북

## 목차
- [[01_개요]] - 스터디 소개 및 멤버
- [[02_Week_01]] - 1주차 종합
- [[02_Week_02]] - 2주차 종합
- ...
- [[03_키워드_사전]] - 주요 키워드 모음
- [[04_프로젝트_제안]] - AI 프로젝트 제안 모음
- [[05_멤버_성장기록]] - 멤버별 학습 여정
```

### 3. 챕터별 파일 생성/업데이트

**개요 챕터** (`_아카이브북/01_개요.md`):
- 스터디 소개, 멤버 목록, 운영 방식

**주차별 챕터** (`_아카이브북/02_Week_0N.md`):
- 해당 주차의 핵심 개념 요약
- 주요 인사이트 하이라이트
- 분석 결과 핵심 내용
- 원본 노트 wikilink 연결

**키워드 사전** (`_아카이브북/03_키워드_사전.md`):
- 전체 주차에서 등장한 주요 키워드
- 키워드별 등장 주차, 관련 멤버, 간단 설명
- 알파벳/가나다 순 정렬

**프로젝트 제안 모음** (`_아카이브북/04_프로젝트_제안.md`):
- 모든 프로젝트 제안을 한 곳에 요약
- 상태 표시 (제안됨/진행중/완료)

**멤버 성장기록** (`_아카이브북/05_멤버_성장기록.md`):
- 멤버별 참여 이력, 관심 키워드 변화, 주요 기여

### 4. 작성 원칙
- 원본 내용을 그대로 복사하지 않고 핵심을 요약 정리
- 원본 노트는 wikilink(`[[원본파일명]]`)로 연결
- 주차가 추가될 때마다 기존 챕터는 유지하고 새 챕터만 추가
- 한국어로 작성, Obsidian 마크다운 형식
