$ARGUMENTS 주차 멤버들의 과제를 바탕으로 링크드인 포스트 초안을 생성해줘.

## 실행 절차

### 1. 데이터 수집
- `00_주차별미션/Week_0$ARGUMENTS/` 의 제출된 멤버 파일 읽기 (10바이트 이하 건너뛰기)
- 각 멤버의 결과물, 만든 과정, 인사이트 섹션 파악

### 2. 멤버별 링크드인 초안 생성

각 제출 멤버마다 `_링크드인초안/Week_0$ARGUMENTS_닉네임.md` 파일을 생성한다.

### 초안 작성 규칙

**톤**: 전문적이지만 캐주얼. 1인칭 시점. 배움을 공유하는 느낌.
**길이**: 800~1200자 (한국어 기준)
**구조**:

```
---
member: 닉네임
week: N
tags:
  - 링크드인
status: draft
---

[후킹 첫 줄 - 질문이나 놀라운 사실로 시작]

---

[본문 - 3가지 포인트로 구성]

1️⃣ 이번 주 뭘 만들었는지 (결과물)
2️⃣ 과정에서 뭘 배웠는지 (인사이트)
3️⃣ 다음에 뭘 해볼 건지 (넥스트)

---

[마무리 CTA - 질문이나 의견 요청]

#AI #ClaudeCode #빌더 #스터디 #AAA
```

### 후킹 첫 줄 예시
- "Claude Code 하나로 하루 만에 웹사이트를 배포했습니다."
- "URL 하나 넣었더니 8개 채널 콘텐츠가 자동으로 나왔습니다."
- "AI에게 답을 구하지 않고, 질문을 던지게 만들었습니다."

### 작성 원칙
- 멤버의 과제 내용을 기반으로 작성 (추측하지 않기)
- 구체적인 숫자와 결과물 포함 ("8채널", "하루 만에", "5장 카드뉴스")
- 배포된 링크가 있으면 포함
- 너무 광고성이면 안 됨 - 배움을 공유하는 톤 유지
- 해시태그 5개 이내

### 3. 결과 안내
생성된 초안 파일 목록을 보여주고, 멤버가 수정 후 링크드인에 게시하도록 안내.
