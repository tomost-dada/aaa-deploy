## 프로젝트 아이디어

### 셀피쉬클럽 통합 콘텐츠 봇
- **작성자**: AI (Week 02 분석)
- **날짜**: 2026-04-01
- **아이디어**: 에밀리의 CRM 자동화 + 오웬의 카드뉴스 생성 + 다니의 멀티채널 콘텐츠를 하나로 합친 통합 봇. "공유회 CRM 시작해줘 {URL}" 한 마디로 알림톡 + 카드뉴스 + SNS 포스트 + 이메일까지 전부 생성
- **왜 좋은가**: 3명이 각자 만든 것을 합치면 셀피쉬클럽 전체 마케팅을 자동화할 수 있음. 이미 각 파트가 검증됨
- **필요한 것**: 에밀리의 에이전트 팀 구조 + 오웬의 Playwright 파이프라인 + 다니의 멀티채널 템플릿

### 멤버별 AI 어시스턴트 커스텀 프레임워크
- **작성자**: AI (Week 02 분석)
- **날짜**: 2026-04-01
- **아이디어**: 흐민의 Sullivan처럼 각 멤버가 자기만의 AI 어시스턴트를 만드는 프레임워크. CLAUDE.md + tone-guide + 에이전트 설계를 템플릿으로 제공하면 비개발자도 자기 업무에 맞는 봇을 만들 수 있음
- **왜 좋은가**: 흐민이 검증한 "형식 없이 생각을 받고, 질문으로 확장하는" 패턴을 다른 도메인에도 적용 가능
- **필요한 것**: Sullivan 설계 원칙 문서화, CLAUDE.md 템플릿, tone-guide 작성 가이드

### 과제 결과물을 바로 재사용 가능한 스킬로 변환
- **작성자**: AI (Week 02 분석)
- **날짜**: 2026-04-01
- **현재 방식**: 멤버가 과제를 하고, 결과물은 각자 레포에만 존재
- **제안하는 방식**: 과제에서 만든 것 중 재사용 가능한 것을 `.claude/commands/`에 스킬로 등록. 다른 멤버도 `/cardnews`나 `/multiuse` 로 바로 사용
- **이유**: 에밀리·오웬·다니가 만든 것은 이미 실전 검증됨. 스킬로 패키징하면 스터디 전체의 생산성이 올라감

### 에이전트 팀 설계 패턴 공유
- **작성자**: AI (Week 02 분석)
- **날짜**: 2026-04-01
- **현재 방식**: 에이전트 팀 구성은 각자 시행착오로 발견
- **제안하는 방식**: 에밀리의 6개 에이전트 팀 구조(director/data-collector/timeline-planner/copywriter/media-ops/dispatcher)를 레퍼런스 패턴으로 문서화
- **이유**: 에이전트 역할 분배는 Week 02의 핵심 학습 포인트. 패턴을 공유하면 다음 과제에서 바로 활용 가능

## 자유 생각

### Week 02 핵심 발견: "자동화의 단위는 채널이 아니라 워크플로우"
- **작성자**: AI (Week 02 분석)
- **날짜**: 2026-04-01
- **내용**: Week 02에서 7명 중 5명이 콘텐츠 자동화를 시도했다. 흥미로운 점은 성공적인 결과물(에밀리, 오웬)의 공통점이 "인스타 자동화"가 아니라 "URL→분석→생성→배포"라는 워크플로우 전체를 자동화한 것이라는 점. 채널 하나를 자동화하는 것보다 워크플로우를 자동화하면 채널 추가가 쉬워진다.

### 비개발자도 Claude Code로 실전 프로젝트를 완성할 수 있다
- **작성자**: AI (Week 02 분석)
- **날짜**: 2026-04-01
- **내용**: 잽은 HTML/CSS/JS만으로 프리미엄 웹 제안서를 하루 만에 완성했고, 비비안은 Webflow→Next.js 전환의 전체 설계를 Claude Code와 함께 9단계로 진행했다. 둘 다 "AI 페어 프로그래밍"의 좋은 사례. 코딩 경험이 적어도 CLAUDE.md를 잘 작성하면 복잡한 프로젝트도 진행 가능.
